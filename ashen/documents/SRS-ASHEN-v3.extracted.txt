Introduction
The purpose of this project is to develop an automated system called ASHEN (Automated Security & Host Exploitation Navigator) that helps cybersecurity professionals and network administrators perform vulnerability scanning, exploitation testing, and remediation using AI-powered analysis.
Traditional vulnerability assessment requires manual execution of tools like Nmap, OpenVAS, and Hydra, which can be time-consuming and error-prone. ASHEN will automate this process and provide a centralized dashboard for reporting and analysis.

Problem Statement
Table 
2
 : Summary of problems and their effects
Category 
Description 
Problem
Manual vulnerability assessment is time-consuming and error-prone; results often need human interpretation and prioritization. Tool outputs are fragmented across scanners and require consolidation.
Affects
Network administrators, security analysts, and organizations needing automated vulnerability checks.
Impact
Inconsistent security testing, delayed detection of threats, and possible exploitation by attackers.
Solution
An integrated platform that automates scan orchestration, validates exploitability in a safe environment, and provides AI-generated remediation steps prioritized by severity along an interactive dashboard.
Business Opportunities
ASHEN can be used by IT companies, security consultants, and network administrators to streamline penetration testing and vulnerability management. It can evolve into a commercial cybersecurity automation tool used in universities, security labs, and IT departments 
Objectives
Following are the points of ASHEN:
To automate vulnerability scanning and exploitation testing.
To integrate tools like Nmap, OpenVAS, and Hydra into a single system.
To provide AI-based recommendations for patching vulnerabilities and suggesting remediation.
To generate detailed and exportable security reports.
To build a secure, user-friendly dashboard for monitoring scans.
Scope
The scope of Automated Security & Host Exploitation Navigator aims to automate vulnerability assessment and exploitation testing within authorized network environments. It integrates tools like Nmap, OpenVAS, Metasploit, and Hydra into a single platform that performs network scanning, vulnerability detection, exploit validation, and AI-based remediation. Through a web dashboard, users can manage scans, select which tests to run, view results, and generate reports. The systemâ€™s AI module analyzes findings and suggests actionable fixes, reducing manual effort and improving security efficiency while maintaining ethical and controlled testing boundaries.
Constraints
Must run tools that require Linux (Kali/Ubuntu).
Requires administrative access to target lab environment
LLM inference may require GPU or host with reasonable CPU/RAM
Legal/ethical constraints limit testing to authorized networks
Stakeholder and User Description
Market Demographics
The target market comprises Organizations and educational institutions seeking automated security assessment tools that can be safely deployed in controlled or authorized environments. It is particularly beneficial for security students, researchers, and small IT teams who require an integrated framework for conducting penetration testing without the need for deep technical expertise or manual tool configuration.
User Environment
Users will access ASHEN through a secure web-based dashboard on desktops or laptops. The interface allows them to initiate scans, monitor progress, and view detailed vulnerability reports.

System Requirements Specification
System Features
ASHEN offers a range of critical features, including:
Frontend
User Management:
 Allows users to securely register, log in, and manage accounts with role-based access control for administrators and analysts
Dynamic Dashboard Interface
: Offers an interactive, web-based dashboard that centralizes all system functionalities allowing users to launch scans, monitor live results, visualize vulnerabilities, and manage reports from a single unified view.
Attack Recommender
: After the vulnerability detection and exploit validation processes, the AI module analyzes the gathered vulnerability data to identify the most likely attack paths and exploit priorities.
AI
-
Based Remediation: 
Once vulnerabilities are validated or successfully exploited, the AI module analyzes the systemâ€™s exposure and recommends actionable fixes to strengthen its defense posture.
Automated Reporting
: Generates detailed and customizable reports in PDF and HTML formats, including vulnerability summaries, severity levels, exploit outcomes, and recommended fixes.
Backend
Host Discovery and Port Scanning
: Automatically identifies active hosts and open ports within the specified network range using advanced scanning techniques.
Vulnerability Detection
: Detects and classifies known vulnerabilities based on CVE data and CVSS scoring standards.
Exploit Validation
: Performs safe and controlled exploitation attempts tests to confirm vulnerability impact.
Functional Requirements
The system must fulfil specific functional requirements, including:
User Management
The system shall allow users to register, log in, and securely manage their accounts.
The system shall authenticate users with role-based access control for administrators and analysts.
Network Scanning
The system shall allow users to input target IP addresses or ranges for scanning.
The system shall use Nmap to identify active hosts, open ports, and detected services.
The scan progress and results shall be viewable in real time through the dashboard interface.
Vulnerability Detection
The system shall initiate OpenVAS scans to detect known vulnerabilities on identified hosts.
The system shall retrieve corresponding CVE entries and assign severity levels using CVSS scoring.
The vulnerabilities and their severity rankings shall be visualized within the dashboard.
Exploit Validation
The system shall safely test detected vulnerabilities using controlled, sandboxed environments and well-known tools such as Metasploit and Hydra.
Planned exploit validation tests include:
FTP brute-force (
vsftpd
)
SSH credential brute-force
Shellshock (CGI) proof-of-concept
MS17-010 (SMB) non-destructive check
All exploit executions will be non-destructive unless explicitly approved by the analyst. Results will be logged and mapped to their corresponding CVEs.
All exploit results shall be recorded and displayed on the dashboard for user review.
Attack Recommender
The system shall use a local LLM engine (e.g., 
Ollama
/
LLaMA
) to analyze scan and exploit data.
The AI shall generate context-aware attack recommendations, such as exploit order, impact likelihood, and required conditions.
The AI shall suggest which vulnerabilities are most critical and should be prioritized for remediation.
Recommendations shall be displayed on the dashboard alongside detected vulnerabilities and exploit results.
AI-Based Remediation
The system shall analyze vulnerability assessment reports to determine feasible remediation strategies.
The AI shall suggest specific configuration changes, patching steps, or system hardening measures relevant to the exploited vulnerability.
The remediation suggestions shall be displayed on the dashboard, linked with each exploit result.
Users shall be able to interact with the AI module (chat or query interface) to obtain clarification or alternate solutions.
Dynamic Dashboard Interface
The system shall provide an interactive, web-based dashboard as the central control panel for all features.
The dashboard shall allow users to start scans, monitor progress, view vulnerabilities, review exploit results, and access AI remediation suggestions.
The dashboard shall include data visualizations such as progress bars, severity charts, and scan summaries for ease of interpretation.
The interface shall support filtering, searching, and exporting of results directly from the dashboard.
Automated Reporting
The system shall generate detailed vulnerability assessment reports in PDF and HTML formats.
Reports shall include network summaries, vulnerability details, exploit validations, AI-based recommendations, and overall risk evaluations.
The reports shall be downloadable through the dashboard and stored in the system for future reference.
Non-Functional Requirements
Performance
The system shall support 10 concurrent scan tasks without affecting responsiveness.
Dashboard updates (for scan results or vulnerability lists) shall appear within 3 seconds of backend completion.
Full report generation for a medium-scale scan (â‰¤500 hosts) shall complete within 120 seconds.
The system shall process and display vulnerability data for up to 1000 hosts per hour per scan worker.
Usability
The interface shall be simple and intuitive, allowing users to initiate scans and view reports easily.
The dashboard shall include clear visual indicators such as progress bars and vulnerability charts.
Security
User credentials and stored data shall be encrypted.
Scanning and exploitation shall be restricted to authorized networks and sandboxed environments.
Reliability
The system shall automatically retry failed scan jobs up to 3 times.
Database backups shall occur every 24 hours, with retention for 90 days.
Upon service restart, incomplete scan sessions shall resume automatically if possible.
The dashboard shall notify users if any worker fails or disconnects.
Scalability
The system shall allow easy addition of new modules or tools without major structural changes.
Ethical Compliance
The system shall display a disclaimer before performing any scan or exploit.
All operations shall comply with authorized and ethical security practices.
Maintainability
The system shall be designed with modular architecture, allowing individual components such as scanning modules, AI engines, and dashboard interfaces to be updated or replaced without affecting other parts of the system.
All source code shall follow clear naming conventions, inline documentation, and version control practices to simplify debugging, updates, and future enhancements.
Compatibility
The system shall operate on Linux (Ubuntu 22.04 or later) as the primary environment.
The dashboard shall be accessible via modern browsers (Chrome, Edge, Firefox).
The backend shall integrate with Nmap, 
Masscan
, Hydra, Metasploit, and OpenVAS via command or API wrappers.
The AI subsystem shall support integration with local LLMs (
Ollama
, 
LLaMA
) and cloud inference (optional).
Reports shall be exportable in PDF, HTML, and CSV formats for interoperability with third-party tools.
AI Governance
All AI-generated outputs (attack recommendations and remediation guidance) are advisory only; no automated remediation actions are applied without human approval.
The system shall record the AI model version, configuration, prompt, and timestamp for every generated suggestion.
AI responses shall pass through safety filters to block destructive or unethical instructions.
Analysts may provide feedback (â€œAccept,â€ â€œReject,â€ â€œRegenerateâ€) to continuously improve guidance accuracy.
AI model updates or retraining must be logged as versioned artifacts under admin supervision.
